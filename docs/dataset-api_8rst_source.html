1 Dataset API
2 =============
3 
4 GET
5 ---
6 
7 GET is used to request data from the data catalog. No authentication is needed.
8 
9 .. code-block::
10 
11  # Get all of the datasets
12  http://dkan/api/v1/dataset
13 
14  # Get a specific dataset by adding the dataset identifier to the end
15  http://dkan/api/v1/dataset/{identifier of your dataset}
16 
17  # Try out the api of a specific dataset
18  http://dkan/dataset/{identifier of your dataset}/api
19 
20 
21 
22 The dataset API has an additional endpoint that breaks out specific sub-elements
23 of the dataset schema (distribution, publisher, theme, and keyword) so that these
24 elements can be worked with as individual objects.
25 Each element is assigned a unique identifier in addition to it's original schema values,
26 this endpoint can be viewed at ``http://dkan/api/v1/dataset?values=both``
27 
28 
29 .. note::
30 
31  POST, PUT, PATCH and DELETE commands require the following:
32 
33  - Basic Auth.
34  - Headers need a **Content-Type** with **application/json**
35

37 Log In
38 ------
39 
40 For actions that require authentication, the user must also have the 'API User' role. This gives the user permission to post, put, and delete datasets through the api.
41 
42 .. code-block::
43 
44  POST http://docker:32774/api/dataset/user/login
45  Content-Type: application/x-www-form-urlencoded
46  Accept: application/json
47 
48  {
49  "username": "{username}",
50  "password": "{password}"
51  }

54 POST
55 ----
56 
57 POST is used to send data to a server to create/update a dataset.
58 
59 The data sent to the server with POST is stored in the request body of the HTTP request
60 
61 .. code-block::
62 
63  POST http://dkan/api/v1/dataset
64  Content-Type: application/json
65  Accept: application/json
66 
67  {
68  "title": "Title of Dataset",
69  "description": "Description for dataset",
70  "identifier": "03ac1e65-bec9-4e36-883d-181c04c95d99",
71  "accessLevel": "public",
72  "bureauCode": ["1234:56"],
73  "@type": "dcat:Dataset",
74  "keyword": [
75  "Keyword One",
76  "Keyword Two",
77  "Keyword Three"
78  ],
79  "contactPoint": {
80  "@type": "vcard:Contact",
81  "fn": "Firstname Lastname",
82  "hasEmail": "mailto:first.last@example.com"
83  },
84  "theme": [
85  "Topic One",
86  "Topic Two"
87  ]
88  }

92 PUT
93 -------
94 
95 PUT is used to update content in your catalog. PUT requests are idempotent, so calling the same PUT request multiple times will always produce the same result. In contrast, calling a POST request repeatedly has the side effect of creating the same content multiple times.
96 
97 .. code-block::
98 
99  PUT http://dkan/api/v1/dataset/{identifier of your dataset}
100  Content-Type: application/json
101  Accept: application/json
102 
103  {
104  "description": "UPDATED description for dataset",
105  "theme": [
106  "Topic Two",
107  "Topic Three"
108  ]
109  }

111 DELETE

113 
114 Delete a dataset
115 
116 .. code-block::
117 
118  DELETE http://dkan/api/v1/dataset/{identifier of your dataset}
119  Content-Type: application/json
120  Accept: application/json
